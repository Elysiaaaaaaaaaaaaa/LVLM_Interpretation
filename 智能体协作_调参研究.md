## iGOS++ 优化原理概述

### integrated_gradient 在做什么

它不是"训练模型"，而是**把 mask 当作可训练参数，对冻结的 LVLM 做梯度计算**。

每调用一次 `integrated_gradient`，内部 `interval_score` 会做三件事：

1. **构造插值路径**：在"全模糊(baseline)"到"mask 完全生效"之间均匀采样 N 步（默认 ig_iter=20），每步生成插值图 `φ(img, blur, α·mask) = α·mask·img + (1-α·mask)·blur`
2. **沿路径算模型置信度**：对每个插值图算 log P(目标词)，取平均
3. **反向传播**：`.backward()` 算出 d(loss)/d(mask)——即**每个 mask 像素变化对模型置信度的影响程度**

梯度方向：
- **梯度 > 0**：让此像素更可见 → 模型更确信 → 重要区域
- **梯度 ≈ 0**：此像素不影响模型 → 不重要

每轮迭代算 4 次 `integrated_gradient`，对应删除(去掉了会挂)和插入(保留了能认)两种互补视角，梯度累积到 `total_grads`，再由 NAG/LS 优化器更新 mask。

## L1/L2/L3 如何影响 mask 更新（iGOS++）

正则项	系数	在干什么	为什么需要
loss_l1	L1	所有像素恒定拉回 1	防止整体 mask 坍缩到 0，设一个基准保留压力
loss_l2	L2	对已经接近 0 的像素猛拉	早期保护——阻止 mask 在前几轮就崩盘，之后衰减消失
loss_tv	L3	相邻像素互相"拉平"	去噪——不让每个像素各自独立决定，强制显著性区域连成片

### 核心机制 — 正则项与 IG 梯度的对抗

iGOS++ 的优化本质是两股力量的**拔河**：

```
IG 梯度（4 次 integrated_gradient）          正则梯度（regularization_loss）
        ↓                                              ↓
  推 mask[d] → 0（标记为"重要"）               推 mask → 1（保留所有区域）
  推 mask[i] → 0（标记为"重要"）
        ↓                                              ↓
  "找到决定性像素"                              "防止过度删除"
```

mask 最终停留在两者的平衡点。正则项的系数 L1/L2/L3 直接控制"保留压力"的强度和方式。

### L1 — L1 稀疏正则（恒幅拉力）

**代码位置**：`IGOS_pp.py:740`

```
loss_l1 = L1 × mean(|1 - mask|)
∂loss_l1/∂mask = -L1 × sign(1-mask) / N   ← 恒幅，与 mask 值无关
```

| 调参方向 | mask 变化 | 热力图表现 |
|---------|-----------|-----------|
| **L1 增大**（如 0.5→2.0） | 每个像素都有更强的恒定拉力推回 1 → 整体更亮 | 显著性区域缩小变淡，对比度降低 |
| **L1 减小**（如 0.5→0.1） | IG 更容易把像素拉向 0 → 整体更暗 | 显著性区域扩大，对比度提升，可能出现过分割 |

**物理意义**：对所有像素**一视同仁**的"保留税"。与 mask 当前值无关，靠近 1 的像素和靠近 0 的像素受到相同的拉力。这使得 L1 是控制整体 mask 亮度的主要旋钮。

---

### L2 — L2 二次正则 + 指数衰减（早期保护，后期释放）

**代码位置**：`IGOS_pp.py:742, 883` | `methods_helper.py:21-22`

```
loss_l2 = L2 × sum((1 - mask)²)
∂loss_l2/∂mask = -2 × L2 × (1-mask)   ← 线性于 (1-mask)
L2_current = L2₀ × exp(-gamma × iter)  ← 每轮指数衰减
```

**衰减曲线（L2₀=0.1, gamma=1.0）：**
| iter | L2 值 | 剩余比例 |
|------|-------|---------|
| 0 | 0.100 | 100% |
| 1 | 0.037 | 37% |
| 2 | 0.014 | 14% |
| 3 | 0.005 | 5% |
| 4 | 0.002 | 2% |
| 5 | 0.0007 | 0.7% |

| 调参方向 | mask 变化 | 热力图表现 |
|---------|-----------|-----------|
| **L2 增大**（如 0.1→0.5） | 前 2-3 轮 mask 几乎不动，被强行按在初始值 0.4 附近 | 延迟收敛，前几轮 loss 无变化 |
| **L2 减小**（如 0.1→0.01） | mask 从第 1 轮就开始自由变化 | 收敛快，但早期可能不稳定 |
| **gamma 增大**（如 1.0→2.0） | L2 衰减更快，保护期缩短 | 等同 L2 减小的效果 |
| **gamma 减小**（如 1.0→0.2） | L2 全程保持一定强度 | 等同 L2 增大的效果 |

**L2 与 L1 的关键区别：**

| 场景 | L1 梯度（恒幅） | L2 梯度（线性） |
|------|---------------|----------------|
| mask=0.9（接近 1） | -L1/N（中等） | ≈ 0（几乎无拉力） |
| mask=0.4（初始值） | -L1/N（中等） | -1.2×L2（中等） |
| mask=0.1（接近 0） | -L1/N（中等） | -1.8×L2（很强） |

→ **L2 精准打击将要塌缩到 0 的区域，而对已经保留的区域几乎不管。** 这是 L2 比 L1 更高效的早期稳定性机制。

**物理意义**：L2 是一个"早期训练轮"——在 IG 梯度还没稳定之前强行拉住 mask 不崩盘，等后期 L2 衰减到接近 0 后就完全放手让 IG 主导。

### L3 — 双边 TV 平滑（空间连续性）

**代码位置**：`IGOS_pp.py:741` | `methods_helper.py:37-62`

```
loss_tv = L3 × bilateral_tv_norm(image, mask)
         = L3 × 0.5 × ( a + b + bil_a + bil_b )

a + b        = 相邻 mask 像素差的平方和（普通TV）
bil_a+bil_b  = 相邻 mask 像素差 × exp(-图像梯度²/σ)（双边TV，在图像边缘处降低约束）
```

| 调参方向 | mask 变化 | 热力图表现 |
|---------|-----------|-----------|
| **L3 增大**（如 10→50） | 相邻像素被迫保持接近，小面积孤立区域被抹平 | 区域连通性好但边缘模糊，像被打过高斯滤波 |
| **L3 减小**（如 10→1） | 每个像素独立优化 | 精细度高但可能出现椒盐噪声 |

**双边特性的效果**：模型判断为"边缘"的地方（原图梯度大），TV 约束自动减弱 → mask 可以在物体边界处锐利变化，不会因平滑而抹掉边界。

**物理意义**：L3 是热力图的"降噪滤波器"。它不让每个像素各自为政，而是强制显著性区域应该连成一片——这符合"重要区域应该是连续的语义区域"这一先验。

### 逐轮动态：三个参数如何协同

```
早期迭代 (i=0~2)
────────────────────────────────────────────────────────
  L2 大（0.10~0.01）→ 防止 mask 塌缩，稳定训练
  L1 恒为 0.5       → 均匀保留压力
  L3 恒为 10        → 约束 mask 空间结构
  IG 梯度被部分抵消  → mask 变化缓慢
  
中期迭代 (i=3~7)
────────────────────────────────────────────────────────
  L2 ≈ 0.005~0      → 几乎消失，不再保护
  L1 = 0.5          → 依旧是 IG 的唯一对抗者
  IG 梯度占主导      → 重要区域 mask 被拉向 0
  mask 格局逐步形成
  
后期迭代 (i=8~14)
────────────────────────────────────────────────────────
  L2 ≈ 0            → 彻底释放
  L1 = 0.5          → 防止过度删除的最后防线
  IG 梯度主导        → 细微调整
  L3 平滑微调        → 消除噪声
```

### 调参基本原则

1. **L1 + L3 决定最终效果的"风格"**：L1 控制亮度，L3 控制平滑度
2. **L2 + gamma 决定"收敛路径"**：只影响前几轮的稳定性，不影响最终结果（只要迭代次数足够）
3. **迭代次数不足时**（如 iterations=5）：L2 全程都有影响 → 需要更小的 L2₀
4. **迭代次数充足时**（如 iterations=15）：L2 只在早期起作用 → 可接受更大的 L2₀

### 常见问题速查

| 现象 | 诊断 | 调整 |
|------|------|------|
| 热力图全红（mask≈1） | L1 过大压死了 IG 梯度 | ↓ L1 |
| 热力图全蓝（mask≈0） | L1 过小，IG 无约束 | ↑ L1 |
| 前 N 轮 loss 完全不动 | L2 过大导致初期冻结 | ↓ L2₀ 或 ↑ gamma |
| 椒盐噪声 | L3 过小，TV 平滑不足 | ↑ L3 |
| 边界模糊 | L3 过大，双边 TV 把边缘也抹了 | ↓ L3 |
| NaN loss | 梯度爆炸 | 检查 lr，检查 pred_probs 中 logits 是否转 fp32 |
